## PASSO A PASSO DE CONFIGURACAO

### FASE 1: Conta Twilio (30 min)

#### 1.1 Criar Conta
```
1. Acesse twilio.com
2. Clique em "Sign Up"
3. Preencha dados (email, senha)
4. Confirme email
5. Adicione telefone para verificacao
```

#### 1.2 Obter Credenciais
```
Console > Account > API Keys

Guardar:
- Account SID: ACxxxxxxxxxxxxxxx
- Auth Token: xxxxxxxxxxxxxxx
```

#### 1.3 Adicionar Creditos
```
Console > Billing > Make a Payment

Minimo recomendado: $20 USD para testes
```

---

### FASE 2: Meta Business (1-3 dias)

#### 2.1 Criar Meta Business Account
```
1. Acesse business.facebook.com
2. Clique em "Criar conta"
3. Preencha dados da empresa
4. Aguarde verificacao (pode levar 1-3 dias)
```

#### 2.2 Verificacao de Negocio
```
Meta Business Suite > Configuracoes > Central de Seguranca

Documentos aceitos:
- CNPJ
- Conta de luz/agua em nome da empresa
- Extrato bancario empresarial
```

---

### FASE 3: Conectar Twilio ao WhatsApp (1 hora)

#### 3.1 No Console Twilio
```
Console > Messaging > Senders > WhatsApp Senders

1. Clique em "Add New Sender"
2. Selecione "Use your own WhatsApp Business Account"
3. Siga o fluxo de conexao com Meta
```

#### 3.2 Vincular Numero
```
1. Informe o numero de telefone (com DDI)
2. Escolha metodo de verificacao (SMS ou Voz)
3. Digite codigo recebido
4. Aguarde aprovacao (pode levar algumas horas)
```

#### 3.3 Verificar Status
```
Console > Messaging > Senders > WhatsApp Senders

Status deve mostrar: "Active"
```

---

### FASE 4: Configurar Voice Calling (30 min)

#### 4.1 Criar TwiML App
```
Console > Voice > Manage > TwiML Apps

1. Clique em "Create new TwiML App"
2. Nome: "WhatsApp Voice Handler"
3. Voice Request URL: https://seu-servidor.com/voice/incoming
4. Salvar
5. Copiar Application SID (APxxxxxxx)
```

#### 4.2 Ativar Calling no Sender

**Via Console:**
```
Console > Messaging > Senders > WhatsApp Senders > Seu Numero

1. Scroll ate "Voice Endpoint Configuration"
2. Selecione o TwiML App criado
3. Salvar
```

**Via API:**
```bash
curl -X POST https://messaging.twilio.com/v2/Channels/Senders/XExxxxxxx \
  -H "Content-Type: application/json" \
  -u $TWILIO_ACCOUNT_SID:$TWILIO_AUTH_TOKEN \
  -d '{
    "configuration": {
      "voice_application_sid": "APxxxxxxx"
    }
  }'
```

## FAZENDO CHAMADAS

### Tipo 1: Chamadas INBOUND (Cliente liga pra voce)

O cliente ve o icone de telefone no chat do WhatsApp e clica.

**Seu webhook recebe:**
```json
{
  "CallSid": "CAxxxxxxx",
  "From": "whatsapp:+5511999999999",
  "To": "whatsapp:+5511888888888",
  "Direction": "inbound"
}
```

**Resposta TwiML:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<Response>
  <Say voice="alice" language="pt-BR">
    Ola! Bem-vindo ao atendimento. Aguarde um momento.
  </Say>
  <Dial>
    <Number>+5511777777777</Number>
  </Dial>
</Response>
```

---

### Tipo 2: Chamadas OUTBOUND (Voce liga pro cliente)

**IMPORTANTE:** Precisa de PERMISSAO do cliente primeiro!

#### Passo 1: Criar Template de Permissao

```bash
curl -X POST 'https://content.twilio.com/v1/Content' \
  -u $TWILIO_ACCOUNT_SID:$TWILIO_AUTH_TOKEN \
  -H "Content-Type: application/json" \
  -d '{
    "friendly_name": "solicitar_chamada",
    "language": "pt_BR",
    "types": {
      "twilio/call-to-action": {
        "body": "Ola! Podemos ligar para voce para continuar o atendimento?",
        "actions": [{
          "type": "VOICE_CALL_REQUEST",
          "title": "Sim, pode ligar"
        }]
      }
    }
  }'
```

**Response:** Guarde o `sid` (HXxxxxxxx)

#### Passo 2: Submeter para Aprovacao Meta

```bash
curl -X POST 'https://content.twilio.com/v1/Content/HXxxxxxxx/ApprovalRequests/whatsapp' \
  -u $TWILIO_ACCOUNT_SID:$TWILIO_AUTH_TOKEN \
  -H "Content-Type: application/json" \
  -d '{
    "name": "solicitar_chamada",
    "category": "UTILITY"
  }'
```

**Aguardar aprovacao:** Pode levar 24-48h

#### Passo 3: Enviar Pedido de Permissao

```bash
curl -X POST "https://api.twilio.com/2010-04-01/Accounts/$TWILIO_ACCOUNT_SID/Messages.json" \
  --data-urlencode "ContentSid=HXxxxxxxx" \
  --data-urlencode "From=whatsapp:+5511888888888" \
  --data-urlencode "To=whatsapp:+5511999999999" \
  -u $TWILIO_ACCOUNT_SID:$TWILIO_AUTH_TOKEN
```

#### Passo 4: Receber Resposta no Webhook

```json
{
  "Body": "VOICE_CALL_REQUEST",
  "ButtonPayload": "ACCEPTED",
  "From": "whatsapp:+5511999999999"
}
```

#### Passo 5: Fazer a Chamada (se ACCEPTED)

```bash
curl -X POST "https://api.twilio.com/2010-04-01/Accounts/$TWILIO_ACCOUNT_SID/Calls.json" \
  --data-urlencode "From=whatsapp:+5511888888888" \
  --data-urlencode "To=whatsapp:+5511999999999" \
  --data-urlencode "Url=https://seu-servidor.com/voice/outbound" \
  -u $TWILIO_ACCOUNT_SID:$TWILIO_AUTH_TOKEN
```

## LIMITES E RESTRICOES

### Chamadas Outbound

| Limite | Valor |
|--------|-------|
| Pedidos de permissao | 1/dia, max 2/semana |
| Chamadas apos aprovacao | Max 5/dia |
| Validade da permissao | 7 dias |
| Chamadas perdidas | 2 consecutivas = pausa |
| Chamadas perdidas total | 4 = revoga permissao |

### Chamadas Inbound

| Limite | Valor |
|--------|-------|
| Chamadas simultaneas | 1.000 por sender |
| Duracao maxima | Sem limite |
| Conexao PSTN | NAO permitido |

## INTEGRACAO COM N8N

### Webhook para Receber Chamadas

```javascript
// n8n Webhook Node
// URL: https://seu-n8n.com/webhook/twilio-voice

// Resposta TwiML
const twiml = `<?xml version="1.0" encoding="UTF-8"?>
<Response>
  <Say voice="alice" language="pt-BR">
    ${$input.item.json.mensagem || "Aguarde um momento"}
  </Say>
  <Dial>
    <Number>${$input.item.json.transferir_para}</Number>
  </Dial>
</Response>`;

return {
  body: twiml,
  headers: {
    'Content-Type': 'text/xml'
  }
};
```

### Fazer Chamada via n8n

```javascript
// n8n HTTP Request Node
// Method: POST
// URL: https://api.twilio.com/2010-04-01/Accounts/{{$env.TWILIO_SID}}/Calls.json

// Body (form-urlencoded):
// From: whatsapp:+5511888888888
// To: whatsapp:{{$json.telefone_cliente}}
// Url: https://seu-n8n.com/webhook/voice-script

// Authentication: Basic Auth
// User: {{$env.TWILIO_SID}}
// Password: {{$env.TWILIO_TOKEN}}
```

## COMPARACAO: TWILIO vs Z-API

| Aspecto | Twilio (Oficial) | Z-API (Nao-oficial) |
|---------|------------------|---------------------|
| **Tipo** | API Oficial Meta | WhatsApp Web |
| **Chamadas Outbound** | ✅ Com permissao | ✅ Direto (15s) |
| **Chamadas Inbound** | ✅ Ilimitado | ✅ Via evento |
| **Risco de Ban** | Baixissimo | Medio |
| **Verificacao** | Obrigatoria | Nao precisa |
| **Setup** | Complexo (dias) | Simples (minutos) |
| **Preco** | Por minuto (~$0.02) | Fixo (R$99/mes) |
| **Selo Verde** | ✅ Sim | ❌ Nao |
| **Templates** | Obrigatorios | Nao precisa |
| **PSTN** | ❌ Nao conecta | ❌ Nao conecta |

### Quando usar cada um?

| Cenario | Recomendacao |
|---------|--------------|
| Volume alto, escala | **Twilio** |
| Marca verificada (selo) | **Twilio** |
| Setup rapido, MVP | **Z-API** |
| Sem verificacao de empresa | **Z-API** |
| Integracao GHL oficial | **Twilio** |
| Custo previsivel | **Z-API** |

## TROUBLESHOOTING

### Erro: "Sender not approved for calling"
```
Causa: Voice calling nao ativado
Solucao: Verificar se TwiML App esta vinculado ao sender
```

### Erro: "Permission denied for outbound call"
```
Causa: Cliente nao deu permissao ou expirou
Solucao: Enviar novo template de permissao
```

### Erro: "Rate limit exceeded"
```
Causa: Muitos pedidos de permissao
Solucao: Aguardar 24h ou usar outro numero
```
