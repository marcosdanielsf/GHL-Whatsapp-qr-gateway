## Z-API - DETALHES

### Planos

| Plano | Preco | Instancias | Msgs | Chamadas |
|-------|-------|------------|------|----------|
| Ultimate | R$ 99,99/mes | 1 | Ilimitadas | ✅ |
| Partner | R$ 54,99-89,99/mes | Multiplas | Ilimitadas | ✅ |

### Endpoint de Chamada

```http
POST https://api.z-api.io/instances/{INSTANCE}/token/{TOKEN}/send-call

Headers:
  Client-Token: {seu-token}
  Content-Type: application/json

Body:
{
  "phone": "5511999999999",
  "callDuration": 5
}
```

| Parametro | Tipo | Obrigatorio | Descricao |
|-----------|------|-------------|-----------|
| phone | string | Sim | Numero com DDI (5511...) |
| callDuration | number | Nao | Duracao em segundos (max 15) |

### Response

```json
{
  "zaapId": "3999984263738042930CD6ECDE9VDWSA",
  "messageId": "D241XXXX732339502B68",
  "id": "D241XXXX732339502B68"
}
```

### Limitacoes

- Duracao maxima do toque: **15 segundos**
- A chamada FAZ O TELEFONE TOCAR, mas a conversa de voz precisa de outro mecanismo
- Nao e possivel gravar ou interceptar o audio da chamada via API

### Funcionalidades de Chamada

2. **Webhooks de chamada** - Recebe eventos de chamadas (missed, rejected, received)
3. **Botoes de chamada** - Adiciona botoes "Ligar" em mensagens

**NAO TEM:** Endpoint para fazer chamada de voz direta como Z-API

## POR QUE OPEN SOURCE NAO SUPORTA?

### Limitacao Tecnica

O protocolo WhatsApp Web (usado por Baileys, WPPConnect, etc) foi projetado para:
- Mensagens de texto
- Midia (imagens, videos, audios gravados)
- Grupos e canais

Chamadas de voz/video usam **protocolo diferente** (WebRTC/SRTP) que:
- Requer conexao peer-to-peer
- Precisa de codecs de audio em tempo real
- Nao foi reverse-engineered completamente

### Status Atual

- **Receber eventos de chamada:** ✅ Funciona (Baileys tem `call` event)
- **Rejeitar chamadas:** ✅ Funciona
- **Fazer chamadas:** ❌ Nao implementado
