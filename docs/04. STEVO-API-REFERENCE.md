## ENDPOINTS DE ENVIO (SEND)

### POST /send/text
Envia mensagem de texto simples.

**Body:**
```json
{
  "number": "+5511999999999",
  "text": "Sua mensagem aqui",
  "delay": 3000,
  "mentionAll": false,
  "mentionedJid": "",
  "formatJid": true,
  "id": "custom-message-id",
  "quoted": {
    "messageId": "xxx",
    "participantJid": "xxx"
  }
}
```

| Campo | Tipo | Obrigatorio | Descricao |
|-------|------|-------------|-----------|
| number | string | Sim | Numero destino (+55...) |
| text | string | Sim | Conteudo da mensagem |
| delay | integer | Nao | Delay em ms antes de enviar |
| mentionAll | boolean | Nao | Mencionar todos no grupo |
| mentionedJid | string | Nao | JID para mencionar |
| quoted | object | Nao | Mensagem para responder |

---

### POST /send/media
Envia midia (imagem, video, documento, audio).

**Body:**
```json
{
  "number": "+5511999999999",
  "type": "image",
  "url": "https://exemplo.com/imagem.jpg",
  "caption": "Legenda da imagem",
  "filename": "documento.pdf",
  "delay": 5000,
  "mentionAll": false,
  "mentionedJid": "",
  "quoted": {}
}
```

| Campo | Tipo | Valores | Descricao |
|-------|------|---------|-----------|
| type | string | image, video, document, audio | Tipo de midia |
| url | string | URL | Link direto para o arquivo |
| caption | string | - | Legenda (imagem/video) |
| filename | string | - | Nome do arquivo (document) |

## ENDPOINT DE CHAMADAS

### POST /call/reject
Rejeita chamada recebida.

**Body:**
```json
{
  "callId": "xxx",
  "callCreator": "5511999999999@s.whatsapp.net"
}
```

**IMPORTANTE:** O Stevo NAO faz chamadas de voz/video. Apenas rejeita chamadas recebidas.

## SCHEMAS IMPORTANTES

### JID (WhatsApp ID)
```
Formato: {numero}@s.whatsapp.net
Exemplo: 5511999999999@s.whatsapp.net
Grupo: {id}@g.us
```

### QuotedStruct (Responder mensagem)
```json
{
  "messageId": "3EB0xxx",
  "participantJid": "5511999999999@s.whatsapp.net"
}
```

## RECURSOS AVANCADOS (via schemas waE2E/waAICommon)

A API inclui schemas para recursos avancados do WhatsApp:

- **AI Rich Responses** - Respostas de IA (Meta AI)
- **Bot Metadata** - Integracao com bots
- **Polls** - Enquetes avancadas
- **Payments** - Pagamentos (Brasil)
- **Newsletters** - Canais
- **Scheduled Calls** - Chamadas agendadas
- **Status/Stories** - Status

Estes schemas indicam que o Stevo usa **whatsmeow** (biblioteca Go) internamente.

---

## LIMITES E BOAS PRATICAS

1. **Rate Limiting:** Use o campo `delay` para evitar bloqueios
2. **Midia:** URLs devem ser publicas e acessiveis
3. **Grupos:** Verifique permissoes antes de acoes admin
4. **Numeros:** Sempre inclua codigo do pais (+55)
5. **Sessao:** Mantenha instancia conectada 24/7
